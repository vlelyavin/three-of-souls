# HANDOFF — для следующей сессии Claude Code

Этот файл — выжимка всего, что нужно агенту, чтобы построить похожий проект
(вебка + жесты + кинематографичные партиклы в браузере) не наступая заново на
грабли, собранные в three-of-souls. Прочитай целиком перед стартом.

---

## 1. Контекст и юзер

- Юзер — Владислав, общение на русском, неформально («бро»). Снимает
  вертикальные ролики в TikTok с этими голограммами.
- **Главный закон: первые две секунды ролика решают всё.** Любая анимация
  появления должна быть эффектной и укладываться в ~2 секунды.
- **Надёжность жеста важнее «умных» защит.** Проверено болью: каждый
  дополнительный guard (поза, скорость, высота руки) убивал срабатываемость,
  юзер требовал откатить. Правильный паттерн: голое условие + кулдаун по
  времени (см. §4). Ложные срабатывания лечи кулдаунами, не позами.
- Если юзер говорит «верни как было» — откатывай **всё и сразу**, без
  частичных компромиссов.
- Идеи и варианты дизайна **обсуждать до** финального плана (он сам просит
  «покреативь, только давай обсудим сначала»).
- Юзер работает на MacBook Air M4. 60 fps — обязательный таргет, партиклы
  легко его роняют (см. §6 про адаптивное разрешение).

## 2. Стек и бутстрап (копируй как есть)

Никакой сборки. Статический http-сервер + CDN importmap:

```html
<script type="importmap">
{ "imports": {
    "three": "https://cdn.jsdelivr.net/npm/three@0.165.0/build/three.module.js",
    "three/addons/": "https://cdn.jsdelivr.net/npm/three@0.165.0/examples/jsm/" } }
</script>
```

MediaPipe руки:

```js
import { HandLandmarker, FilesetResolver } from
  'https://cdn.jsdelivr.net/npm/@mediapipe/tasks-vision@0.10.14';
// FilesetResolver.forVisionTasks('.../tasks-vision@0.10.14/wasm')
// modelAssetPath: storage.googleapis.com/mediapipe-models/hand_landmarker/hand_landmarker/float16/1/hand_landmarker.task
// delegate: 'GPU', runningMode: 'VIDEO', numHands: 2, confidence ~0.4
```

Есть и Pose/Face Landmarker в том же пакете tasks-vision — для проектов с
телом (сесть/отодвинуться в кресле) бери `PoseLandmarker` тем же способом.

Рендер: WebGLRenderer `powerPreference:'high-performance'`, pixelRatio
`min(devicePixelRatio, 1.5)`. Постобработка: EffectComposer (**HalfFloatType
обязательно**) → RenderPass → UnrealBloomPass → финальный ShaderPass с
**ACESFilmicToneMapping в шейдере** (не в renderer'e, иначе блум клипует).

На этой машине **node сломан** (brew/simdjson) — скриптовые правки файлов
делай через python3, сервер — `python3 -m http.server`.

## 3. Архитектурный скелет (переиспользуй)

4 файла, разделение обязанностей:

- `gestures.js` — треки рук поверх MediaPipe (вся грязь трекинга тут);
- `tree.js` (контент) — процедурная генерация геометрии частиц; у каждой
  частицы несколько «домов»-позиций в атрибутах (aTreePos, aExpandPos,
  aScatterDir…), переходы между ними — **униформами в GLSL**, JS каждый кадр
  двигает только числа-таргеты: `value += (target - value) * k * dt`;
- `fx.js` — блум, орбы, ударные волны, молнии, жилы;
- `main.js` — стейт-машина + детекторы жестов поверх треков + рендер-луп.

Стейт-машина — плоский switch по `state`, переходы задокументированы. В
three-of-souls: HIDDEN → SUMMON → GROWING → TREE ⇄ EXPANDED, TREE → GRABBED →
RIPPED → DEAD → RESURRECT. После каждого перехода запоминай `*At = now` —
кулдауны строятся на этих метках.

Кинематографика:
- **отдельные «кино-часы»**: `cine.timeScale` плавно тянется к таргету,
  слоу-мо = `timeScale 0.5` на 450мс в момент удара; весь визуал живёт на
  cine-времени, трекинг рук — на реальном;
- **cap скорости раскрытия**: reveal двигается не быстрее 0.5/с — каскад
  роста всегда ~2с, каким бы резким ни был триггер;
- ударный ритм: тёмная пауза ~0.6с между вспышкой-имплозией и взрывом роста —
  дешёвый способ сделать «дорого».

## 4. Жестовый движок — самое ценное

Паттерны, без которых жесты «работают раз в 10 попыток»:

1. **Сшивка треков по предсказанной позиции.** Быстрый свайп улетает далеко
   между детекциями и иначе спавнит новый (неподтверждённый) трек:
   `px = tr.x + tr.vx * min(dt, 0.2)`, матчить детекцию к ближайшему
   предсказанию, порог ~0.5 норм. координат.
2. **Адаптивное EMA**: `a = min(0.5 + jump * 3, 0.9)` — дрожь фильтруется
   жёстко, размашистое движение проходит почти сырым.
3. **Свайпы — по смещению из истории, не по мгновенной скорости.** Кольцо
   `tr.hist` (700мс), детектор берёт сэмпл 450мс назад и сравнивает с текущим.
   EMA душит скорость — на скорости свайпы никогда не срабатывали стабильно.
4. **Классификация ладонь/кулак с гистерезисом** (open > 1.35, fist < 1.12,
   openness = средняя дистанция кончиков от запястья / размер ладони) **+
   подтверждение 4 кадра** → события justOpened/justClosed.
5. **Дедуп детекций**: MediaPipe видит одну руку дважды (порог 0.22) и
   выдаёт «призраков» с низким score — фильтруй score < 0.7.
6. **Триггеры жестов — голые + кулдауны.** Финальная форма самого главного
   жеста (кулак → ладонь = призыв):
   `if (h.justOpened && now - hiddenAt > 1200) doSummon()` — и ВСЁ. Никаких
   проверок позы/скорости/высоты. Ложный призыв после свайпа-скрытия лечится
   именно кулдауном `hiddenAt`.
7. Полезные метрики на треке: `pinchIdx` (большой↔указательный / размер),
   `pinch` (большой↔средний, для щелчка), `ang`/`angVel` (наклон кисти,
   поворот), `prevPalmDur` (сколько держалась предыдущая поза).
8. Щелчок пальцами: прайм = pinch<0.4 ≥2 кадров при полураскрытой ладони,
   выстрел = pinch>1.0 при скорости разлёта >5/с в окне 260мс + кулдауны.
9. **Потеря руки — не событие.** Grace-период 350–800мс до реакции (рука
   мигает у края кадра); удержание чего-то в руке при пропаже руки → мягкий
   возврат объекта «домой» с задержкой ~1.2с.

## 5. GLSL-грабли и рецепты

- **UnrealBloomPass сбрасывает разрешение при `composer.setSize`** — в
  resize-хендлере после него заново `bloom.setSize(w/2, h/2)`. Иначе
  фуллскрин = ~9× стоимость блума, fps умирает незаметно.
- **Компактный яркий источник + блум = квадратное гало** (усечённая
  сепарабельная гауссиана в грубых мипах). Не лечится радиусом. Рецепт:
  притушить ядро (alpha ~0.2 у центра), а свечение дать отдельным круглым
  гало-биллбордом (`exp(-d*d*4.5)` на квадрате-плоскости).
- Тёмная «полость» вокруг яркого объекта в гуще частиц — лучший способ
  сделать его читаемым: `vColor *= 1 - glow * 0.8 * exp(-dist²*2.2)` для
  соседних частиц. Не накручивай яркость объекта — приглушай фон.
- Эффект «увядания» с оседанием — **клампи по Y** (`max(..., floorY)`), иначе
  дерево уезжает под пол с каждым циклом.
- Якорные точки внутри масштабируемой группы — бери matrixWorld **самой
  внешней** группы (баг: якорь через внутреннюю матрицу игнорил масштаб).
- Счётчик-нормализатор: если группа скейлится, а эффект должен быть
  инвариантен (разлёт галактики) — передавай `uNorm = 1/scale` униформой.
- Вебкам-фон рисуй **внутри WebGL** (quad с текстурой видео: зеркало, cover,
  приглушение `uDim`, лёгкий сдвиг в холод) — тогда он проходит через блум и
  голограмма «вживается». Живая яркость на хоткеях + localStorage.
- Режим блэкаута (спрятать вебкам-quad, хоткей B): юзер накладывает запись
  экрана на видео с телефона блендом **Screen** — чёрное исчезает. Это его
  основной пайплайн съёмки, закладывай с самого начала.

## 6. Производительность

- Бюджет: ~100к частиц на M4 тянет, но с блумом на пределе. Адаптивное
  разрешение: ступени renderScale [1.5, 1.25, 1.0, 0.8], прогрев 3с, вниз
  при fps<52, вверх при fps>58 подряд ×4.
- Каждая частица — один `gl_PointSize`-спрайт, весь моушен в вершинном
  шейдере. JS не трогает атрибуты после инициализации, только униформы.
- pixelRatio cap 1.5, блум на половинном разрешении — визуально не отличить.

## 7. Тестирование без камеры (headless)

Force-параметры состояния (обязательно заложи в новый проект с первого дня —
это единственный способ итерировать визуал без рук перед камерой):

```
?r=0..1 &m=0..1 &death=0..1 &nova=0..1 &heart=0..1 &orbx &orby
?state=tree|expanded &debug=1 &ui=0 &cleartest=1
```

Скриншот:

```bash
"/Applications/Google Chrome.app/Contents/MacOS/Google Chrome" --headless=new \
  --window-size=1600,1000 --virtual-time-budget=15000 \
  --screenshot=/tmp/shot.png "http://127.0.0.1:8471/index.html?r=1&ui=0"
```

Консоль: добавь `--enable-logging=stderr` и грепай. Для теста с «камерой»:
`--use-fake-ui-for-media-stream --use-fake-device-for-media-stream`.
При force-параметрах прячь лоадер жёстко (`style.display='none'`, transition
opacity не успевает к скриншоту).

Клавиши-дублёры каждого жеста (T/H/E/R/N…) — тоже с первого дня, и с русской
раскладкой (`k === 'r' || k === 'к'`).

## 9. Следующий проект: «сердце» (про любовь)

Идея юзера: сидит в кресле → жест пальцами → **рисуется красивое сердце из
частиц** → отодвигается на кресле → **материализует лук в руках** →
натягивает тетиву → **стреляет в сердце**. Стрельба из лука двумя руками —
это pinch задней руки (натяжение = расстояние между руками, отпускание
pinch = выстрел); для «отодвинулся в кресле» смотри PoseLandmarker или
масштаб ладони как прокси дистанции.

Идеи-кандидаты (НЕ реализовывать молча — юзер хочет сначала обсудить):
рисование контура сердца пальцем с последующим «доведением» до идеальной
параметрической кривой; сердце бьётся, при приближении руки — чаще; выстрел
со слоу-мо и стрелой-кометой; взрыв попадания в тысячи лепестков/маленьких
сердец, осыпающихся как снег; вариант «стрела сшивает две половинки»;
«поцелуй в ладонь и сдуть» — частицы складываются в сердце; два человека в
кадре рисуют по половинке; финал — частицы складываются в текст (имя/дата).

Стартуй с копии скелета этого репо (gestures.js переносится без изменений,
fx.js почти без изменений), выкинь tree.js и пиши новый контент-модуль.
